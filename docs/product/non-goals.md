# Non-Goals

1. **Not another coding agent.** Ultimate Harness coordinates and standardizes agent work; it does not need to be the model/runtime itself.
2. **Not a BMAD clone.** BMAD-style roles are useful, but Ultimate Harness must remain broader than BMAD.
3. **Not a shell-script wrapper.** Runtime adapters need structured state and lifecycle reporting, not just `command: string`.
4. **Not an unbounded autonomous agent.** Human checkpoints, verification evidence, and promotion policy are core.
5. **Not a docs-only framework forever.** Documentation comes first, but the design should lead to a CLI and adapter implementation.
6. **Not a replacement for GitHub/Linear.** It should link to issue trackers rather than own all planning data.
7. **Not a sandbox monoculture.** Git worktrees and AgentFS have different strengths; the harness should model both.
