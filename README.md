# Ultimate Harness

Ultimate Harness is an early-stage project to build a unified software-development harness that gathers the strongest ideas from several specification-driven development (SDD), agent workflow, and skill systems.

The goal is to combine proven patterns from existing frameworks into a practical "super harness" for planning, specifying, executing, verifying, and safely iterating on agentic software work.

## Inspiration

This project will study and integrate ideas from:

- [Specsafe](https://github.com/Agentic-Engineering-Agency/specsafe/issues) for specification safety and issue-driven development practices.
- [BMAD Method](https://github.com/bmad-code-org/BMAD-METHOD) for structured agent roles, planning, and delivery workflows.
- [superpowers](https://github.com/obra/superpowers) for composable agent capabilities.
- [GSD](https://github.com/gsd-build/get-shit-done) for disciplined execution workflows and durable project context.
- [matt-pocock/skills](https://github.com/mattpocock/skills) for focused, reusable development skills.
- [oh-my-openagent](https://github.com/code-yeongyu/oh-my-openagent) for open agent ecosystem patterns.
- [OpenSpec](https://github.com/Fission-AI/OpenSpec) for open specification workflows.

## Core Runtime Direction

Ultimate Harness will explore using [oh-my-pi](https://github.com/can1357/oh-my-pi) or [Pi](https://pi.dev/) as the core project engine. The intent is to provide a powerful foundation for orchestrating agent workflows while preserving clear project state, reproducible execution, and inspectable outcomes.

## Sandboxing Focus

A major focus of this project is safe execution through sandboxing. Ultimate Harness will evaluate and incorporate patterns from [agentfs](https://github.com/tursodatabase/agentfs/blob/main/MANUAL.md) to isolate agent work, manage filesystem effects, and make agent-driven changes easier to inspect, approve, revert, or promote.

## Project Vision

Ultimate Harness aims to provide:

- Specification-first planning and execution.
- Reusable skills and agent capabilities.
- Structured workflows for research, implementation, testing, and review.
- Sandboxed environments for safer autonomous development.
- Clear audit trails for decisions, file changes, and verification results.
- A practical bridge between SDD frameworks, agent orchestration, and production-ready engineering workflows.

This repository is currently a starting point for that exploration.
