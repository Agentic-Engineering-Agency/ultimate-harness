## Lalo — agent platform builder

Needs a runtime-agnostic harness that can unify Codex, Claude Code, Pi, Hermes, and future tools without losing specs, skills, or audit trails. Values proactive structure, deep docs, reusable skills, and verification-first workflows.

## Implementation agent

Receives a mission packet and must know exactly what to do, what context to read, which skills to apply, which files may change, which checks to run, and how to report progress.

## Runtime adapter author

Builds an adapter for Codex, Claude Code, Pi, Hermes, or another runtime. Needs a clear lifecycle contract, conformance expectations, and examples.

## Reviewer / maintainer

Approves or rejects sandbox output. Needs diffs, test results, spec compliance notes, security/sandbox analysis, and traceability to issue/spec/mission.

## Skill author

Creates reusable procedures that can be selected for missions. Needs a skill format with metadata, body, prerequisites, verification hooks, and related skills.
